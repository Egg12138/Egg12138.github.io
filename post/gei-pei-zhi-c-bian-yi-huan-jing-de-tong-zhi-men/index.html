WSL下的Linux用gcc:

在shell界面：
sudo apt-get install gcc等待安装
vim是自带的，
写代码和编译和运行
vim filename.c
按[I]进入insert：
[Esc]--[:][wq]w保存，q保存更改后退出编辑界面
gcc 刚刚的filename.c这样会编译位a.out文件，肯定不太方便，所以：
输这个：
gcc 刚刚的filename.c -o newname 比如gcc hw.c -o hw就把刚刚写的名为hw.c的源代码编译输出成hw文件，存放在同一个目录下， 也可以放别的地方，所以这个newname,filename其实不仅仅指文件名，更应该路径。
编译成功就不会报错，失败会报错，然后继续vim filename.c去debug，
成功后运行：
如果在当前路径下：./newname即可运行

在Windows下，Powershell中

其实一样用gcc等命令进行，
需要配置的是环境变量
MinGW-w64 zip 解压缩，找到bin文件夹的位置，这个是多数windows程序都要有的核心的文件夹，gcc也在此目录下，
配置正确环境变量（程序位置）之后，处在C盘某位置的cmd/powershell就可以访问到其他路径的命令gcc了。如果不配置这个也是可以运行的，比如gcc在D:\Shaw\gcc\g++.exe，那我再命令行输入D:\Shaw\gcc\g++.exe也可以运行gcc，但是这样太麻烦，而且不像是“命令”。
赋值\bin位置后，就到环境变量--Path--Edit--Add --[Paste]
之后再cmd下：gcc -v看一下能否正常。
结束
